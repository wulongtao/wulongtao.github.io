Hexo搭建个人网站

分类： Hexo
标签： Hexo

日期：2023-07-02 11:25:56

准备工作

git账号、nodejs环境

如果时win11，要检查下下面这个配置要打开

github配置

  1. 创建一个名字为 如下格式的仓库

  1. 然后再里面加一个index.html的文件，内容如下
    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>xxiaohei</title>
</head>
<body>
    <h1>小小黑的个人主页</h1>
    <h1>Hello ~</h1>
</body>
</html>

  1. 再setting页面中找到page的菜单做如下配置


配置完成后点击页面可以访问到你的网页就证明配置成功了！

Hexo安装和配置

安装hexo

npm install -g hexo-cli

查看版本

hexo -v

创建项目 xxiaohei-website

hexo init xxiaohei-website
cd .\xxiaohei-website\
npm install

本地启动

hexo g
hexo server

启动后访问成功

更换主题（重点）

上面配置配置完成后，进入项目的theme目录，执行如下命令

git clone git@github.com:Aizener/hexo-theme-cola.git

修改下面两个配置
关闭highlight配置项，打开prismjs配置

把theme配置改为：hexo-theme-cola主题：

同一个配置文件改下url地址

安装完成后，执行以下命令重新生成主题，并开启服务。

hexo clean 
hexo g 
hexo s

启动完成已经可以看到新的主题了

安装好主题后，还需要新建几个页面，分别执行以下命令：

  1. 新建日志页面：

    hexo new page log

  2. 新建友链页面：

    hexo new page link

  3. 新建个人页面：

    hexo new page about

  4. 新建分类页面：

    hexo new page categories

  5. 新建标签页面：

    hexo new page tags

  6. 新建工具页面：

    hexo new page tools

    新建完成页面之后，就可以到主题的_config.yml配置文件配置对应的信息了。（页面根据是否需要来新建）

Live2D动画挂件配置

使用如下指令安装Live2D模块、注意需要在博客的根目录通过终端执行

npm install --save hexo-helper-live2d

如果不想使用、想彻底卸载，命令如下：

npm uninstall hexo-helper-live2d

安装完成之后，就会在博客的根目录package.json文件中存在依赖

在博客根目录的node_moduels文件夹下可以看到安装的Live2D模块

参考这个下面的网站，使用命令去下载
https://github.com/xiazeyu/live2d-widget-models

npm install {packagename}

npm install live2d-widget-model-nico

安装完成后，打开根目录的_config.yml配置文件，加入如下配置

# Live2D
## https://github.com/xiazeyu/live2d-widget.js
## https://l2dwidget.js.org/docs/class/src/index.js~L2Dwidget.html#instance-method-init
live2d:
  model:
    scale: 1
    hHeadPos: 0.5
    vHeadPos: 0.618
    use: live2d-widget-model-nico # 下载的动画模型名称
  display:
    superSample: 2
    width: 120
    height: 200
    position: left # 模型在网页显示位置
    hOffset: 20
    vOffset: 50
  mobile:
    show: true  # 移动设备是否显示
    scale: 0.5
  react:
    opacityDefault: 0.7
    opacityOnHover: 0.2

然后

hexo clean 
hexo g 
hexo s

部署

运行如下命令安装依赖

npm install hexo-deployer-git --save

修改_config.yml目录下的信息

deploy:
  type: git
  repo: https://github.com/wulongtao/wulongtao.github.io.git
  branch: main
  token: token自行获取

使用如下命令部署到github

hexo g -d

发布文章

如下修改 Hexo 博客目录中的 _config.yml，打开这个配置是为了在生成文章的时候生成一个同名的资源目录用于存放图片文件。

post_asset_folder: true

如下命令创建文章

hexo new post 测试文章

这三种写法在md文件中图片是无法显示的，但是在页面上能正常显示。
图片的引入方式可参考官方文档 https://hexo.io/zh-cn/docs/asset-folders.html，有详细介绍。
